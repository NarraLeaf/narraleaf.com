NarraLeaf Project 是一个面向视觉小说体验构建的生态系统，覆盖从创作到交付的主要路径。它将零代码 Studio、桌面应用工具链和轻量 React 播放器放在同一套项目理念下，让团队可以选择合适的起点，同时保留进入完整 NarraLeaf Project 生态的空间。

这个项目围绕轻量 Web 技术、结构化内容和可增长的工作流设计。个人创作者可以先专注于场景与叙事流程，开发者也可以继续使用 TypeScript、版本控制、打包流程和应用级定制能力。

## 项目理念

NarraLeaf Project 不只是产品列表，也是一种让视觉小说项目在规模增长后仍然保持清晰的项目结构。

这个生态强调：

- 将创作工具、运行时行为、渲染层和播放器集成分离清楚。
- 建立在 Web 与 TypeScript 基础上，便于接入已有开发工作流。
- 让项目文件和工具链可以像现代软件一样被审查、版本管理、测试和交付。
- 提供多种入口，但让它们服务于同一个项目方向，而不是变成互不相关的产品。

因此，一个项目可以从可视化工作流开始，在需要时深入到桌面交付细节，也可以只把播放器层嵌入已有的 React 产品中。

## 主要路径

### NarraLeaf Studio

[NarraLeaf Studio](/zh/docs/studio) 是零代码创作与编辑路径。它面向希望通过引导式可视化环境构建故事结构、场景和制作流程的创作者与团队。

当项目的首要任务是创作和维护内容，而不是立即处理应用内部实现时，Studio 是最合适的起点。

### NarraLeaf Desktop

[NarraLeaf Desktop](/zh/docs/narraleaf/library) 是桌面应用工具链，覆盖用于构建和交付 NarraLeaf Desktop 应用的 runtime、renderer 与 CLI。

这条路径面向需要掌控完整应用的开发者，包括本地开发、打包、运行时集成和交付流程。

### NarraLeaf-React

[NarraLeaf-React](/zh/docs/narraleaf-react) 是轻量 React 播放器路径。它适合希望将视觉小说体验嵌入 Web 产品，或需要深度定制播放器 UI 的团队。

这条路径让播放器保持贴近 React 生态，适用于故事体验只是更大应用中的一部分的场景。

## 如何选择

如果你的首要目标是创作故事内容、管理场景，并通过可视化流程推进制作，请从 [NarraLeaf Studio](/zh/docs/studio) 开始。

如果你要构建完整的桌面视觉小说应用，并需要访问 runtime、renderer、CLI 和交付工作流，请查看 [NarraLeaf Desktop](/zh/docs/narraleaf/library)。

如果你已经有 React 应用，需要一个轻量播放器，或希望完全掌控周围的 UI 与产品体验，请查看 [NarraLeaf-React](/zh/docs/narraleaf-react)。

这些路径是同一项目生态中的兼容组成部分。团队可以从其中一条路径开始，并随着项目变得更技术化、更协作化或更接近发布，逐步采用其他部分。

## 生态方向

NarraLeaf Project 也在围绕素材打包、音频、UI 组合、脚本工具和制作工作流发展更多支持工具。这些工具的目标是服务三条主要路径，而不是制造新的复杂入口。

长期来看，NarraLeaf Project 希望形成一套连贯的视觉小说项目工具集：对创作者足够易用，对开发者足够实用，并且能支撑团队维护和交付真实应用。
