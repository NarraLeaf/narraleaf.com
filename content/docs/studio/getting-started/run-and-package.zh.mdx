Studio 提供两种不同的输出流程：开发模式用于制作期间检查，构建发行版本用于生成可分发文件。**导出项目**则用于传递 Studio 项目本身，不是游戏安装包。

## 在开发模式中运行

可以通过以下任一入口启动开发模式：

- 在 UI 编辑器中选择画布右上角的播放按钮，从当前页面或游戏 UI 开始。
- 选择**开发 > 开发模式**，运行项目的默认入口。
- 使用顶部操作栏中的开发模式按钮。

开发模式会打开独立窗口。对 UI 文档或蓝图作出修改后，可重新加载运行内容；运行日志、节点跟踪和宿主 API 调用会显示在调试工具或控制台中。

再次选择**开发模式**可停止当前会话。

<Callout type="info">
编辑器画布不执行完整游戏逻辑。涉及页面切换、故事推进、控件事件或存档时，应以开发模式中的结果为准。
</Callout>

## 构建发行版本

<Steps>
<Step>

### 打开构建窗口

选择**开发 > 构建发行版本**，或使用顶部操作栏中的构建按钮。
</Step>
<Step>

### 选择目标

选择平台、格式和架构。可用目标取决于当前系统：

- Windows 产物可从 macOS、Windows 或 Linux 构建。
- macOS 产物必须在 macOS 上构建。
- Linux 产物可在 macOS 或 Linux 上构建。
- Web 产物可在任意受支持的桌面系统上构建。

移动端目标仍处于开发阶段，不应作为当前发行流程的依据。
</Step>
<Step>

### 检查标识与内容

确认版本号、产品名、应用 ID、版权和图标。版本号使用语义化版本格式，例如 `1.0.0`。

构建窗口还会列出随包插件、语言、网络策略和资源保护状态。Web 构建始终以可部署的静态文件发布，资源保护对 Web 产物不生效。
</Step>
<Step>

### 选择输出目录

选择一个可写目录，并根据需要设置压缩级别。若目录中已有同名产物，构建可能覆盖旧文件。

产物列表会在构建前显示预计生成的文件名。
</Step>
<Step>

### 开始构建

处理所有阻断项后选择**开始构建**。构建在后台进行，进度和详细信息显示在控制台中。
</Step>
</Steps>

## 构建前检查

构建窗口会区分阻断项和警告：

- 无目标、版本号无效或输出目录不可写会阻止构建。
- 缺少图标时仍可构建，但会使用默认 Electron 图标。
- 桌面产物目前不会自动完成代码签名。玩家首次打开时可能看到系统安全提示。
- 跨平台桌面构建可能需要首次下载对应的 Electron 文件。

## 导出 Studio 项目

需要把可编辑项目交给其他成员或归档时，选择**文件 > 导出项目**。Studio 会生成 `.nlspkg` 项目包。接收方在启动器中选择**导入项目**，并将其解包到一个空目录。

`.nlspkg` 包含项目内容，但不等同于发行构建。玩家不需要 Studio 项目包。
