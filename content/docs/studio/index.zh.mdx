NarraLeaf Studio 是用于制作视觉小说的桌面编辑器。故事、角色、素材、界面和发行设置都保存在同一个 NarraLeaf 项目中；常规制作不要求编写外部脚本。

<Callout type="warning">
Studio 仍处于早期开发阶段。界面名称和项目格式可能随预览版本调整。重要项目应使用版本控制或保留独立备份。
</Callout>

## Studio 中的项目

一个 Studio 项目包含以下几类内容：

- **故事**：以故事、章节、场景和动作组织剧情。
- **角色**：管理角色资料、形态、差分和关联素材。
- **素材**：导入并分组管理图片、音频及其他项目文件。
- **页面**：标题、设置、存档、历史、画廊等完整屏幕。
- **游戏 UI**：对话框、选项、通知、NVL 和舞台 HUD 等游玩界面。
- **蓝图**：为页面、控件和游戏 UI 添加交互与运行逻辑。
- **本地化**：维护源语言、翻译和审校状态。

Studio 会把这些内容写入项目目录。关闭工作区时会保存当前修改；项目目录仍应作为一个整体移动、备份或提交到版本控制。

## 页面与游戏 UI

两者都在 UI 编辑器中制作，但用途不同。

| 类型 | 用途 | 尺寸 |
| --- | --- | --- |
| 页面 | 标题页、设置页、存档页等完整界面 | 创建时单独指定 |
| 游戏 UI | 对话、选项、通知、NVL 或舞台内容 | 使用项目的舞台尺寸 |

游戏 UI 按位置挂载。对话、选项、通知和 NVL 等位置在同一项目中各只能有一个活动界面。

## 编辑与运行

工作区中的画布用于排版和静态检查。需要验证真实交互、蓝图执行或宿主 API 时，应在**开发模式**中运行当前界面或项目。准备交付给玩家时，再使用**构建发行版本**生成桌面或 Web 产物。
