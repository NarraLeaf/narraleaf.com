The Studio API is the `narraleaf-studio/plugin` surface: everything a plugin can reach in the **editor process** (the workspace window). It is where a plugin contributes to Studio itself — panels, actions, keybindings, blueprint node metadata, widgets, language packs — and where it reads project assets and its own storage.

This is the editor counterpart to the [Host API](/docs/studio/plugin/host-api), which acts on a running game. A plugin's `studio` entry talks to the Studio API; its `runtime` entry talks to the game. See [The two entries](/docs/studio/plugin/create-first-plugin).

## definePlugin

```ts
import { definePlugin } from "narraleaf-studio/plugin";

export default definePlugin({
    setup(app) {
        // register contributions, read services
        return () => { /* optional cleanup */ };
    },
});
```

`setup(app)` runs when the workspace window loads the plugin. It may be async. Return a cleanup to run on unload — or return nothing, since the host tracks every registration anyway.

## The `app` object

| Property | Type | What |
| --- | --- | --- |
| `app.plugin` | `PluginIdentity` | Identity: `id`, `name`, `version`, `publisher`. |
| `app.manifest` | `NormalizedPluginManifestV2` | The normalized manifest. |
| `app.services` | `PluginServices` | The curated editor API — the service families below. |
| `app.privileged` | `BoundPrivilegedFacade` | Elevated capabilities (filesystem, bash) gated by manifest permissions. |

## Registration conventions

These hold across every service that registers something.

- **`register` returns a cleanup.** Each `register` / `registerMany` returns a `PluginCleanup` (`() => void | Promise<void>`) that removes exactly what it registered. `registerMany` returns one cleanup for the whole batch.
- **Unload reclaims everything.** The host tracks each registration, so unloading the plugin removes it even if you never call the returned cleanup. Cleanups are idempotent.
- **Ids are namespaced.** Every id or type you register must be prefixed with your plugin id (`yourname.plugin.thing`). An unprefixed id throws.
- **Imperative calls return their value, not a cleanup.** `ui.editors.*`, `ui.notifications.*`, `i18n.format*`, and `blueprintNodes.notifyDynamicSelectOptionsChanged` are actions.
- **Blueprint node registration returns `void`.** Node definitions are session-persistent — removing one would orphan nodes in open documents — so there is nothing to dispose.

<Callout type="warn">
Plugins are not sandboxed. The Studio API is a curated whitelist, but `app.privileged` reaches the real filesystem and shell once the manifest declares the matching permission. Everything runs with Studio's own access.
</Callout>
