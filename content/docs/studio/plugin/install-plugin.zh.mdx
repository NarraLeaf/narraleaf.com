---
title: 安装插件
description: 从文件夹安装插件、批准其权限，并启用、禁用或移除它。
---

Studio 从本地文件夹安装插件——也就是包含插件 `manifest.json` 和构建出的入口文件的那个文件夹。发布的插件以该文件夹的 zip 形式分发；先解压，再安装。

<Steps>
<Step>

### 获取插件

下载插件的发布包（一个 `.zip`）并解压。你应当得到一个在顶层包含 `manifest.json` 的文件夹：

```
hello-1.0.0/
  manifest.json
  main.js
  runtime.js
```

</Step>
<Step>

### 从文件夹安装

打开启动器，进入 **Plugins**，选择 **Install from folder**。选中解压后的文件夹（含 `manifest.json` 的那个）。Studio 会校验清单，并把插件复制到它自己的存储中。

</Step>
<Step>

### 批准权限

如果插件声明了提权权限——文件系统或 API 能力——Studio 会弹出提示，逐项列出它请求的内容。审查后批准或拒绝。没有声明任何权限的插件会直接安装，不弹出提示。

</Step>
</Steps>

<Callout type="warn">
插件不在沙箱中运行。一旦安装并启用，插件就拥有与 Studio 相同的访问权限，其游戏侧代码会运行在你玩家的机器上。只从你信任的来源安装插件，并在批准前阅读权限提示。
</Callout>

## 启用、禁用与卸载

启动器中的 **Plugins** 列表会显示所有已安装的插件。在那里你可以：

- **启用 / 禁用**——被禁用的插件仍保持已安装状态，但不会加载。它的编辑器贡献项和游戏侧代码都会停止。切换会在下一次工作区或游戏加载时生效。
- **卸载**——删除插件的文件。

提供**语言包**的插件在启用时会把它的语言加入 设置 → 语言；禁用或卸载它会移除这些语言。如果它提供的正是你当前使用的语言，Studio 会回退到英文；重新启用该插件后，你的选择会恢复。

## 出问题时

如果插件的清单无效、入口文件缺失，或在加载时抛出错误，Studio 会在插件列表中把它标记为错误状态，并在那里显示相应信息。加载失败的插件绝不会阻止 Studio 启动，一个损坏的插件也不会影响其他插件。

如果项目用到的某个蓝图节点或控件没有可用的提供方——插件缺失、被禁用，或某个版本不再声明它——预览和导出会失败，并给出一条指明该插件和类型的诊断信息，而不是发布一个带有无效节点的游戏。
