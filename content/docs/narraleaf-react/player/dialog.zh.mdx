---
title: "对话框"
---
`Dialog` 组件用于渲染对话框，包括角色名牌、对话文本以及可选的**对话框头像**（小型立绘头像）。

可通过 `game.configure({ dialog })` 传入自定义组件。你可以组合 `Avatar`、`Nametag`、 `Texts`，或使用 [`useAvatar`](/zh/docs/narraleaf-react/player/hooks/useAvatar)、[`useDialog`](/zh/docs/narraleaf-react/player/hooks/useDialog) 处理布局逻辑。

<Callout type="info">
    默认头像槽位可直接使用 `Avatar`。如果对话布局需要根据是否存在头像而变化，请使用 `useAvatar()`。
</Callout>

## 快速示例

1. 导入组件

```tsx
import { Dialog, Texts, Nametag, useGame } from "narraleaf-react";
```

2. 组合默认 ADV 布局（可选 `Avatar`）

```tsx
import { Avatar, Dialog, Nametag, Texts } from "narraleaf-react";

function GameDialog() {
    return (
        <Dialog className="bg-white">
            <div className="dialog-content flex items-start gap-4 w-full h-full">
                <Avatar />
                <div className="dialog-text-content min-w-0 flex-1">
                    <Nametag className="font-bold" />
                    <Texts className="text-lg" />
                </div>
            </div>
        </Dialog>
    );
}
```

内置的 `DefaultDialog` 采用相同结构：横向排列 `Avatar`，右侧一列放置 `Nametag` 与 `Texts`。

3. 在游戏中注册

```tsx
function App() {
    const game = useGame();

    useEffect(() => {
        game.configure({
            dialog: GameDialog,
        });
    }, []);

    return /* ... */
}
```

## 组件

### Dialog

渲染对话框容器。子节点通常为 `Avatar`、`Nametag`、`Texts`（布局可自定义）。

- `children?: React.ReactNode` — 对话框内容。
- `...props: HTMLMotionProps<"div">` — 传给内部的 `motion.div` 包裹元素。除了普通 div props 外，也支持 `initial`、`animate`、`exit`、`transition`、`layout` 以及 Motion 事件处理器。

外层 dialog wrapper 会继续负责宽高比缩放以及点击/按键处理。视觉 transform 应直接写在 `Dialog` 上，播放器会把它们应用到内部 motion 元素。

已导出的 props 类型：`NametagProps`、`TextAppearanceProps`、`TextsProps`、`RawTextsProps`、`EntryTextsProps`。

### Nametag

渲染说话人名字。需放在 `Dialog` 内。

`Nametag` 现在由组件 props 驱动：通过 props 控制内容与展示。未传自定义内容时，会回退到当前说话人名称；未传 `color` 时，会在可用时回退到当前角色颜色。

- `character?: Character | null` — 显式指定角色来源。
- `entry?: NvlDialogEntry` — NVL 条目来源。
- `name?: React.ReactNode` — 覆盖显示名称。
- `color?: Color` — 名牌文字颜色。
- `children?: React.ReactNode` — 完全自定义名牌内容。
- `...props: Omit<React.HTMLAttributes<HTMLDivElement>, "children" | "color">`

### Texts

渲染逐字显示的对话正文。需放在 `Dialog` 内。

文本默认外观来自组件 props 或继承的 CSS。`Sentence` 与 `Word` 的局部样式仍会覆盖对应行或词。

- `children?: never`
- `defaultColor?: Color` — 当句子/词没有设置颜色时使用的回退文本颜色。
- `fontSize?: React.CSSProperties["fontSize"]`
- `fontWeight?: React.CSSProperties["fontWeight"]`
- `fontWeightBold?: React.CSSProperties["fontWeight"]` — 句子/词为粗体时使用的字重。
- `fontFamily?: React.CSSProperties["fontFamily"]`
- `...props: React.HTMLAttributes<HTMLDivElement>`

### Avatar

根据当前台词解析并渲染对话框头像；若当前行没有头像则**不渲染任何节点**（不占位）。

- `...props: React.ImgHTMLAttributes<HTMLImageElement>` — 会与默认样式合并（见 [useAvatar](/zh/docs/narraleaf-react/player/hooks/useAvatar)）。

默认图片样式：

- `width` / `height`：`96`
- `objectFit`：`"cover"`
- `borderRadius`：`6`
- `flex`：`"0 0 auto"`

## Dialog motion props

自定义对话框组件可以直接向 [`Dialog`](/zh/docs/narraleaf-react/player/dialog) 传入 Motion props。内置的 `DefaultDialog` 默认不会添加进入或退出动画。

```tsx
import { Dialog, Nametag, Texts } from "narraleaf-react";

function GameDialog() {
    return (
        <Dialog
            initial={{ opacity: 0, y: 20 }}
            animate={{ opacity: 1, y: 0 }}
            exit={{ opacity: 0, y: -20 }}
            transition={{ duration: 0.25, ease: "easeOut" }}
            layout
        >
            <Nametag />
            <Texts />
        </Dialog>
    );
}
```

presence 时序由播放器处理：

- 对话到对话的直接替换会保留同一个可视 dialog slot。上一句不会播放 `exit`，下一句也不会只因为文本或 nametag 变化就重新播放进入动画。
- 对话后接非对话内容时，当前对话会从 active text list 中移除，因此可以继续播放 `exit`，同时剧情向后推进。
- 如果旧退出动画还没结束，后面又出现新对话，新对话会获得新的 presence key。旧退出和新进入可以重叠播放。
- 通过 dialog slot 渲染的菜单 prompt 会在选项出现前使用同一套 presence 行为。
- 正在退出的对话框处于 inactive 状态：点击、skip、auto-forward 都会被忽略，保留节点也不能重复调用 `onFinished`。

`GameConfig.animationPropagate` 会传给 dialog 的 `AnimatePresence` 边界。除非你的自定义对话框内部还嵌套了需要传播退出动画的 `AnimatePresence`，否则保持 `false` 即可。

## 对话框头像

对话框头像用于在 ADV 文本框旁显示小头像，适用于：说话人在台上、画外音、以及大多数台词**不需要**逐句配置头像的场景。

### 设计模型

- **`Character` 承载头像意图** — 默认/画外音头像，以及立绘绑定。
- **舞台上的 `Image` 可细化头像** — 仅当该立绘为当前绑定的**可见**肖像时生效。
- **`Sentence` 只覆盖例外** — 隐藏头像或单行特殊资源/解析器。

解析顺序：

1. 旁白或无名角色 → 无头像。
2. `sentence.config.avatar === false` → 无头像。
3. 句子级头像覆盖（字符串、解析器等）。
4. 该角色当前**最近可见**的绑定立绘的头像。
5. 角色级默认头像。
6. 否则无头像。

引擎**不会**自动从全身立绘裁切头像。未配置头像则不显示。

### 角色级头像

当同一张头像应同时用于台上台词与画外音时使用。

```ts
import { Character } from "narraleaf-react";

const alice = new Character("Alice", {
    avatar: "/assets/alice/avatar-default.png",
});

alice.say("画外音时也可以使用角色级头像。");
```

链式写法：

```ts
const alice = new Character("Alice")
    .setAvatar("/assets/alice/avatar-default.png");

alice.say("本句使用 Alice 的默认头像。");
```

### 绑定舞台立绘

当可见立绘应决定对话框头像时，将舞台 `Image` 绑定到角色。

```ts
import { Character, Image } from "narraleaf-react";

const aliceBody = new Image({
    name: "alice-body",
    src: "/assets/alice/body-normal.png",
    opacity: 1,
});

const alice = new Character("Alice", {
    avatar: "/assets/alice/avatar-default.png",
    portraits: [
        {
            image: aliceBody,
            avatar: "/assets/alice/avatar-normal.png",
        },
    ],
});
```

当 `aliceBody` 在当前场景可见时使用立绘头像，否则回退到角色级头像。

```ts
const alice = new Character("Alice")
    .setAvatar("/assets/alice/avatar-default.png")
    .addPortrait(aliceBody, {
        avatar: "/assets/alice/avatar-normal.png",
    });
```

### 基于标签表情的头像

立绘头像可以是**解析函数**，入参包含当前立绘、`currentSrc`、`tags`、`character`、`sentence`、`gameState`。返回值可以是：

- 图片 URL 或 `StaticImageData`；
- `null` — 在本层强制不出现头像；
- `undefined` — 交由下一层回退逻辑处理。

```ts
const aliceBody = new Image({
    name: "alice-body",
    src: {
        groups: [
            ["normal", "happy", "angry"],
            ["school", "casual"],
        ],
        defaults: ["normal", "school"],
        resolve: (emotion, outfit) => `/assets/alice/body-${emotion}-${outfit}.png`,
    },
    opacity: 1,
});

const alice = new Character("Alice", {
    avatar: "/assets/alice/avatar-default.png",
    portraits: [
        {
            image: aliceBody,
            avatar: ({ tags }) => {
                const emotion = tags?.[0] ?? "normal";
                return `/assets/alice/avatar-${emotion}.png`;
            },
        },
    ],
});
```

### 单行覆盖

```ts
alice.say("仅本行隐藏头像。", {
    avatar: false,
});

alice.say("本行使用特殊切图。", {
    avatar: "/assets/alice/avatar-special.png",
});

alice.say("本行使用解析器。", {
    avatar: ({ tags, currentSrc }) => {
        if (tags?.includes("angry")) {
            return "/assets/alice/avatar-angry-close.png";
        }
        return undefined;
    },
});
```

除 `avatar: false` 外，句子级覆盖优先于立绘与角色默认。

### 多个可见立绘

若同一角色多个绑定立绘同时可见，引擎按场景中展示顺序选取**最近显示且仍可见**的一个。`opacity` 等效为 0 的显示对象会被忽略。

## 自定义对话框与 `useAvatar`

简单布局直接使用 [`<Avatar />`](/zh/docs/narraleaf-react/player/hooks/useAvatar)。若网格/弹性布局依赖「是否有头像」，请使用 [`useAvatar()`](/zh/docs/narraleaf-react/player/hooks/useAvatar)（`visible`、`src`、`character`、`portrait` 等）。

详见 [useAvatar](/zh/docs/narraleaf-react/player/hooks/useAvatar)。

## TypeScript 参考（公开导出）

类型与类由 `narraleaf-react` 导出（并见 [Character](/zh/docs/narraleaf-react/core/elements/character)、[SentenceConfig](/zh/docs/narraleaf-react/core/types/SentenceConfig)、[CharacterConfig](/zh/docs/narraleaf-react/core/types/CharacterConfig)）：

- `DialogAvatarSource`、`DialogAvatarResolverContext`、`DialogAvatarResolver`、`DialogAvatar`
- `CharacterPortraitConfig`、`DialogAvatarResolution`
- `Character` 方法：`setAvatar`、`addPortrait`、`setPortraits`
- `Sentence` / `say` 的 `avatar`：`SentenceUserConfig` 参见 [SentenceUserConfig](/zh/docs/narraleaf-react/core/types/SentenceUserConfig)
- React：`Avatar`、`useAvatar`，类型 `DialogAvatarContext`

默认 UI 的头像解析在播放器内部完成；通常只需配置角色与句子，再渲染 `<Avatar />` 或调用 `useAvatar()`。

## 自定义对话框外观

### 示例：容器样式

```tsx
function GameDialog() {
    return (
        <Dialog
            style={{
                backgroundColor: "rgba(0, 0, 0, 0.5)",
                borderRadius: "10px",
                padding: "20px",
            }}
        >
            {/* ... */}
        </Dialog>
    );
}
```

### 示例：名牌装饰

```tsx
function GameDialog() {
    return (
        <Dialog>
            <Nametag
                style={{
                    backgroundImage: "url('/path/to/image.png')",
                    backgroundSize: "cover",
                    width: "100%",
                    height: "100%",
                }}
            />
        </Dialog>
    );
}
```
