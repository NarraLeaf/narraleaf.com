钩子是一种向游戏中注入自定义逻辑的方式。它对于创建插件非常有用。

```typescript
game.hooks.hook("init", () => {
    console.log("Game initialized");
});
```

## 公共方法

### hook

<Steps>
<Step>

添加一个钩子到游戏中。

```typescript
const token = game.hooks.hook("init", () => {
    console.log("Game initialized");
});

return function cleanup() {
    token.cancel();
}
```

- `key: Hooks` - 钩子的键。
- `hook: (...value: T[K]) => VoidFunction | void` - 返回一个清理函数的钩子函数。
- 返回 `LiveGameEventToken` - 参见 [LiveGameEventToken](/zh/docs/narraleaf-react/core/types/LiveGameEventToken)
</Step>
</Steps>

### unhook

<Steps>
<Step>

从游戏中移除一个钩子。

```typescript
const listener = function () { /* ... */ };

game.hooks.hook("init", listener); // 添加一个钩子

game.hooks.unhook("init", listener); // 移除钩子
```

- `key: Hooks` - 钩子的键。
- `listener: (...value: T[K]) => VoidFunction | void` - 被添加到钩子中的监听函数。
</Step>
</Steps>

## 钩子列表

### init

<Steps>
<Step>

此钩子在游戏初始化时且播放器渲染前被调用。

```typescript
game.hooks.hook("init", () => {
    console.log("let's do something cool here");
});
```
</Step>
</Steps>

### preloadImage

<Steps>
<Step>

此钩子在预加载图片时被调用。这对于创建图片代理逻辑很有用。

- `src: string` - 图片的源地址。
- `set: (src: string, options?: RequestInit) => void` - 设置图片源地址的函数。

```typescript
game.hooks.hook("preloadImage", (src, set) => {
    const newSrc = `https://example.com/proxy?url=${src}`;
    set(newSrc, {
        credentials: "include",
    });
});
```
</Step>
</Steps>

### beforeRestore

<Steps>
<Step>

此钩子在反序列化游戏状态之前被调用。

```typescript
game.hooks.hook("beforeRestore", () => {
    console.log("准备恢复游戏状态");
});
```
</Step>
</Steps>

### afterRestore

<Steps>
<Step>

此钩子在反序列化游戏状态之后被调用。

```typescript
game.hooks.hook("afterRestore", () => {
    console.log("游戏状态恢复成功");
});
```
</Step>
</Steps>
