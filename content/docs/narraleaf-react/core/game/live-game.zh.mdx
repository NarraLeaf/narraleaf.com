### deserialize

<Steps>
<Step>

加载保存的游戏

调用此方法后，当前游戏状态将丢失，舞台将触发强制重置

> **注意：**即使只更改一行脚本，保存的游戏也可能与新版本不兼容

示例：

```typescript
const savedGame = {
    // ...保存的游戏数据
};

// 在组件中使用钩子
const {game} = useGame();

// 将保存的游戏数据传递给游戏实例
game.getLiveGame().deserialize(savedGame);
```

- `savedGame: SavedGame` - [SavedGame](/zh/docs/narraleaf-react/core/types/SavedGame)
</Step>
</Steps>

### getHistory

<Steps>
<Step>

获取游戏历史。这个方法用于创建回溯。

自存档格式 v2 起，回溯历史会被持久化，因此 `deserialize` 之后本方法会立即返回完整历史 —— 读档后的游戏不再从空回溯开始。

```typescript
import { useGame, GameHistory } from "narraleaf-react";
```

```typescript
const game = useGame();
const history = game.getHistory();

function handleUndo(history: GameHistory) {
    game.undo(history.token);
}

return (
    <div>
        <h3>回溯</h3>

        {history.map((item) => (
            <div
                key={item.token}
                onClick={() => handleUndo(item)}
            >
                {/* show the action text */}
                {/* text is available when the action is "say" or "menu" */}
                {item.element.text}
            </div>
        ))}
    </div>
);
```

- 返回 `GameHistory[]` - 游戏历史，参见 [GameHistory](/zh/docs/narraleaf-react/core/types/GameHistory)
</Step>
</Steps>

### undo

<Steps>
<Step>

撤销游戏动作。

```typescript
game.undo(history.token);
```

- `token?: string` - 历史项的令牌，参见 [GameHistory](/zh/docs/narraleaf-react/core/types/GameHistory)。如果未提供，则撤销最后一个动作。

> **注意：** `undo` 依赖仅在游戏运行期间存在的内存撤销历史。读档之后它已无内容可回退 —— 此时请使用 [restoreToHistory](#restoretohistory) 回到已读档游戏中的某一行。
</Step>
</Steps>

### restoreToHistory

<Steps>
<Step>

将游戏恢复到回溯历史中的某个过去行。

与 [undo](#undo) 不同，本方法在**读档之后**依然有效：每个回溯条目都携带一份自包含的快照，恢复时会重新应用该快照并把回溯裁剪回该行，而不依赖内存中的撤销历史。

```typescript
const history = game.getHistory();

// 跳回过去的某一行 —— 包括从已读档游戏中恢复出来的行。
game.getLiveGame().restoreToHistory(history[0].token);
```

- `token: string` - 要恢复到的历史项令牌，参见 [GameHistory](/zh/docs/narraleaf-react/core/types/GameHistory)。
- 返回 `boolean` - 若成功恢复该行则为 `true`；若令牌未知或该条目没有恢复快照则为 `false`。
</Step>
</Steps>

### waitForRouterExit

<Steps>
<Step>

等待路由退出。

此方法可以用于在创建新游戏时等待路由退出。

```typescript {10}
const {game} = useGame();
const router = useRouter();
const liveGame = game.getLiveGame();

useEffect(() => {
    router.clear().cleanHistory();

    const token = liveGame
        .newGame()
        .waitForRouterExit()
    
    token
        .promise
        .then(() => {
            dispatchState({ isPlaying: true });
        });

    return () => {
        token.cancel();
    };
}, []);
```

- 返回 `{ promise: Promise<void>; cancel: VoidFunction; }`
</Step>
</Steps>

### fastForward

<Steps>
<Step>

将游戏快进到下一个菜单（或故事结尾）。

其间的每一行都会被真实执行，因此回溯历史及其恢复快照会像正常游玩一样累积 —— 只是更快且静音。快进期间音频被静音，进行中的过渡立即结算，定时暂停（`Control.sleep`、自动前进）也会立即解决。一旦有菜单在等待选择，它就会停下，因此选择本身始终留给玩家。由于历史全程累积，[getHistory](#gethistory) 和 [restoreToHistory](#restoretohistory) 会像正常游玩一样覆盖被快进的区间。

```typescript
// 快进到下一个决策点。
await game.getLiveGame().fastForward();

// 或运行到故事结尾。
await game.getLiveGame().fastForward({ until: "end" });
```

- `options.until?: "menu" | "end"` - `"menu"`（默认）在下一个菜单处停止；`"end"` 运行到故事结束。
- `options.maxSteps?: number` - 前进步数的安全上限（默认为 `maxStackModelLoop` 配置）。
- 返回 `Promise<{ reason: "menu" | "end" | "maxSteps" }>` - 停止的原因。
</Step>
</Steps>
