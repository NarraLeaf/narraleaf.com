---
title: "控制"
---
`控制`是一个类，它具有一些用于流程控制的实用方法。

```typescript
Control.do([
    character1.say("hello"),

    // 播放声音并同时摇动图像
    Control.allAsync([
        sound.play(),
        shake(image1),
    ]),
]);
```

## 静态方法

<Steps>
<Step>

### do

按顺序执行操作，等待每个操作完成

- `actions: ActionStatements` - [ActionStatements](/zh/docs/narraleaf-react/core/types/ActionStatements)
- 返回 `ChainedControl` - 链式控制实例
</Step>
</Steps>

<Steps>
<Step>

</Step>
</Steps>

### repeat

<Steps>
<Step>

多次执行操作

- `times: number` - 次数

</Step>

### whileLoop

<Steps>
<Step>

当条件为真时执行操作

```typescript
const loopState = new Persistent("loop-state", {
    tries: 0,
});

story.registerPersistent(loopState);

scene.action([
    loopState.set("tries", 0),

    Control.whileLoop(loopState.evaluate("tries", (tries) => tries < 3), [
        character.say`第 ${loopState.get("tries")} 次尝试`,
        loopState.set("tries", (tries) => tries + 1),
    ]),
]);
```

- `condition: Lambda<boolean> | LambdaHandler<boolean>` - 要检查的条件
- `actions: ActionStatements` - [ActionStatements](/zh/docs/narraleaf-react/core/types/ActionStatements)
- 返回 `ChainedControl` - 链式控制实例
</Step>
</Steps>

### breakLoop

<Steps>
<Step>

打破当前循环（repeat/while）
只能在循环体内使用

```typescript
scene.action([
    Control.whileLoop(loopState.evaluate("tries", (tries) => tries < 10), [
        Condition.If(loopState.equals("tries", 3), [
            character.say("到这里停止。"),
            Control.breakLoop(),
        ]),
        loopState.set("tries", (tries) => tries + 1),
    ]),
]);
```

- 返回 `ChainedControl` - 链式控制实例
</Step>
</Steps>

### waitForClick

<Steps>
<Step>

暂停执行，直到用户在舞台任意位置点击（不包括对话框、按钮、菜单等 GUI 元素）。类似于在 [Sentence](/zh/docs/narraleaf-react/core/elements/character/sentence) 中插入无时长的暂停。

适用于在 ADV 模式中创建「点击继续」的停顿，或在 NVL 块内等待玩家准备好后再继续。

```typescript
scene.action([
    character.say("请仔细阅读..."),
    Control.waitForClick(),
    character.say("我们继续。"),
]);
```

- 返回 `ChainedControl` - 链式控制实例
</Step>
</Steps>

### label

<Steps>
<Step>

在当前场景内标记一个命名的跳转点，供 [`jump`](#jump) 跳转。label 在运行时不可见——它只是直接跳过到下一个动作。

label 的名称仅在其所在场景内有效，因此不同场景可以复用相同的名称。在同一场景内重复声明同一名称会导致构建失败。

```typescript
scene.action([
    Control.label("intro"),
    character.say("开始吧。"),
]);
```

- `name: string` - label 名称，在其所在场景内唯一
- 返回 `ChainedControl` - 链式控制实例
</Step>
</Steps>

### jump

<Steps>
<Step>

跳转到**同一场景**内的某个 [`label`](#label)，并从那里继续播放。

与 [`Scene.jumpTo`](/zh/docs/narraleaf-react/core/elements/scene#jumpto) 不同，它停留在当前场景内——不会卸载或重新初始化任何场景，只移动播放头。目标必须是同一场景内声明的 label，否则故事将构建失败。

`Control.jump` 会重定向主故事流程，因此应将它放在分支的最后一个动作（例如菜单选项）。若要循环一个场景，请通过菜单或条件来驱动循环，而不是从 `repeat` / `while` 循环体中跳出。

```typescript
scene.action([
    Control.label("start"),
    Menu.prompt("再来一次？")
        .choose("是", [Control.jump("start")])
        .choose("否", []),
]);
```

- `name: string` - 目标 label 的名称，需在同一场景内声明
- 返回 `ChainedControl` - 链式控制实例
</Step>
</Steps>
